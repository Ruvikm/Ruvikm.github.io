搭建Rasa机器人

发表于2024-10-31|更新于2024-11-18|毕设笔记
|字数总计:3.5k|阅读时长:11分钟|阅读量:

开始整毕设了，虽然没有开题，但有一些进度还是好的，就记录一下自己做毕设的工作吧，目前做的是毕设的第一块也是比较重要的一块部分，搭建一个Rasa机器人

Rasa UI 开发笔记

nodejs + angularjs + express

该rasa-ui前端项目以sqlite数据库（/server/data/db.sqlite3）进行开发，数据库初始化等维护在 /server/db/db.js 文件中，启动时会根据方法 checkDBSchema() 查询 version 表判断是否需要新建数据库和表（createDBSchema方法创建表）。如果想添加新的表，再 createDBSchema() 方法中填写建表语句，然后修改version 表的信息或删除version 表，否则表不会被创建

添加新的模块：

数据库操作层

在 /server/db/ 目录下参照除 db.js 外的其他文件编写自定义模块的数据库操作js文件

给1中的方法添加路由

在 /server/routes/index.js 文件中，首先使用const faq = require('../db/faq')引入 1 中自定义的数据库js文件，然后使用router.get("/faq", faq.getFaqs);、router.put('/faq', faq.updateFaq);等方法将路由path与自定义的数据库操作函数绑定，注意这里使用的是通过 path + http method 即 访问路径+get或post或put或delete 的方式来定位最终访问的方法

接口注册

在 /web/src/app/services/api.js 文件中使用 app.factory('Name', function ($resource, appConfig) {}); 方法进行注册，其实是通过 $resource(appConfig.api_endpoint_v2 + '/faq', {id: '@id'}); 方法进行注册的，第一个参数即访问路径，第二个参数即访问的默认参数意义不大，这里注意的是第三个参数，要想使 update 方法能够在 put 方式下能调用 update 语句，需要追加第三个参数：{update: {method: ‘PUT’}}

在左侧菜单栏添加自定义菜单

在 /web/src/app/shared/sideber.html 文件中添加自定义菜单项

添加自定义菜单模块的 html 页面

在 /web/src/app/components 目录下创建模块目录，然后添加相应的 html 和 js 文件，大家可以参照其他模块来编写这两个文件，这里需要注意的是如果调用 update 时无法效用相应的数据库 update 操作请再仔细按照步骤 3 在操作一下，还有要将 js文件追加到 /web/src/index.html 中

添加自定义页面的路由

在 /web/src/app/router.js 文件中的 .otherwise({ redirectTo: '/' }) 前追加插入 .when(path, {}） 即可，这里注意 controller 的值要与步骤 5 中 js 里定义的保持一致， templateUrl为自定义的 html 路径，activePage 要与步骤 4 中的 $route.current.activePage 保持一致

项目运行端口以及rasa端口维护

直接 npm start 启动

项目端口：/server/server.js 58行 const listener = server.listen();

Rasa端口：package.json 34行 rasa_endpoint:

Docker容器：

项目端口：docker-compose.yml 20行

Rasa端口：docker-compose.yml 9行和24行

或

项目端口：Dockerfile 11行

Rasa端口：Dockerfile 12行

项目接口访问

http://ip:port/api/v2/ + 步骤 2 中注册的path 与 httpMethod(get,post,put,delete)

具体可参照步骤 3 中 $resource 方法的第一个参数—-其中 appConfig.api_endpoint_v2(/api/v2) 在 /server/server.js 文件的第35行使用方法 app.use 注册，在 /web/src/app/config/app.config.js 文件中定义为全局变量

Rasa UI 说明书

管理系统地址

系统账号密码

首页看板介绍

登陆进首页数据看板显示的是请求处理总数，日志总数，当前加载的训练模型文件以及最近30天的请求处理数曲线图

机器人管理

点击进入机器人管理，右上角+按钮添加聊天机器人，添加好的直接点击机器人名称可进行编辑

新增机器人

进入机器人添加页面填写机器人名称（建议使用英文名称），机器人配置—主要用于聊天训练的配置可直接复制已添加的机器人配置，服务输出目录—训练好的模型文件输出的目录位置（建议填写models），填写好后点击保存即可

编辑机器人

保存好新增的机器人后，回到机器人管理界面点击刚刚新添加的机器人进入机器人编辑页面添加其他训练数据。点击右上角的删除按钮可删除当前机器人，所有与此机器人相关的数据都将被删除，但训练好的模型文件可保留下来。点击下面意图、实体、近义词和正则表达式面板右侧的 + 按钮分别添加相关数据，点击添加的相关数据进行编辑

添加意图

输入意图名称—建议使用英文名称，点击保存进行保存

意图就是用于在聊天中判断用户的聊天意图，方便对话进行

修改意图

在机器人编辑页面点击新增的意图进入意图编辑页面进行修改

表述即为用户可能说的话，输入表述点击回车进行添加，如输入有误可点击表述列表最右侧的删除按钮删除后重新添加

如果表述中涉及到实体（插槽），可使用鼠标选中表述中实体可能的值，然后点击右侧第二按钮进行关联，然后点击第三按钮设置这个实体相关的属性，如果表述中涉及多个实体值要标记，请按从后往前的顺序依次进行标记和设置—-在设置实体前请先在机器人编辑页面添加实体相关数据

点击右侧第一个按钮可将当前表述模拟聊天进行发送并返回各意图的命中概率

新增实体

输入实体名称（建议使用英文名称），选择实体类型（建议选择text，数值类型选择float）

编辑实体

机器人编辑页面点击新添加的实体名称进入实体边界页面，点击右上角删除按钮删除实体—删除或修改前请先确认该实体是否在使用

新增近义词

输入近义词词源点击保存进行添加

修改近义词

机器人编辑页面点击近义词列表的词进入近义词编辑页面，输入词源的近义词—至少输入三个字符不足使用逗号补全，点击回车会生成标签进行添加，点击标签的❌号，可删除近义词，点击最右侧删除按钮会删除词源以及所有近义词

添加正则表达式

输入正则表达式名称—建议从实体名称中选择，然后输入正则表达式点击保存进行添加

修改正则表达式

机器人编辑页面点击正则表达式名称进入正则表达式编辑页面，点击右上角删除按钮可删除该正则表达式

回复管理

由表单forms维护和actions维护，主要用于维护对用户的回复

首先在机器人下拉列表中选择需要维护的机器人

添加表单

点击表单面板右侧的+按钮进入新增表单界面，输入表单名称点击保存进行添加

编辑表单

删除表单

点击新增的表单的最右侧的删除按钮删除该表单，于此表单相关联的数据也一并被删除

添加表单插槽

点击新增表单最右侧的+按钮添加插槽，插槽是从配置的实体中选择的，请在下拉框中选择合适的实体作为插槽，然后点击右侧的 对号 按钮保存进行添加

修改表单槽值

重选实体插槽，点击右侧对号按钮进行保存，点击删除按钮进行删除

添加actions

点击Actions右侧的+按钮进入新增action界面，输入action名称—名称规则请使用 utter_意图名称 然后点击保存进行添加

编辑actions

删除actions

点击新增的action最右侧的删除按钮删除该action

新增回复

点击action最右侧的+按钮新增回复文字，选择回复类型并填写回复文字点击右侧 对号 按钮保存，点击删除按钮可删除此条回复，如果回复中涉及实体插槽可使用 {实体名称} 进行占位，系统会自动识别用户话术中相应的插槽值进行填补

故事管理

主要用于维护用户与系统间可能出现的对话流程

新增故事

点击故事面板右侧+按钮添加故事

编辑故事

删除故事

点击新添加的故事的最右侧删除按钮可删除该故事

添加故事情节

点击故事右侧下三角按钮展开故事添加故事情节，右侧搜索框可搜索前面添加的意图、实体、表单和Actions—搜索出来的列表左侧是名称右侧是类型，点击其中一项可自动将情节按格式添加到左侧文本框中—每个故事限一个故事流程情节，多个请追加故事进行添加，也可进行手动调整，编辑好后点击保存进行添加

模型训练

用于将前面的各配置进行训练生成模型文件用于机器人与用户之间的交流

首先在机器人下拉列表中选择需要训练的机器人

下方的标签页中会整合显示出选中的机器人在前面所作操作的各项数据，可以手动进行更改，点击强制更新开关按钮后在此处做的更改就会失效被更新为存库的数据，可添加备注然后点击开始训练，此时请不要离开此页面，数据量越大驯良时间就越长，请耐心等待，训练完成后会给出是否训练成功的提示，然后就可以在模型列表中看到训练好的模型了

模型列表

主要是用于查看当前加载的聊天模型详细信息和在机器人下拉列表中选中的机器人的所有已经训练好的模型的列表

点击卸载模型多轮智能对话将不可用，请谨慎操作。

点击模型列表的删除按钮会将已训练好的模型文件删除。

点击模型列表右侧第一个按钮就会切换加载这个模型文件，智能多轮对话将会以这个模型展开

UI设置

该UI会定时监控后台连接状态，此处功能即设置监测的时间间隔，也可以禁用监测
